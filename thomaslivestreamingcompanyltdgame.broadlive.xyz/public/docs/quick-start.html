🎯 Quick Start Guide

Set up the app-based game access system in just 5 minutes!

Step 1: Edit Environment Configuration

Open your .env file and add:

BD_GAME_FINAL_ALLOWED_APP_PACKAGES=com.jambo.app,com.example.games
BD_GAME_FINAL_DEVELOPER_PASSWORD=#jambo

Replace:

  • com.jambo.app with your actual app package name
  • #jambo with your secure developer password

Step 2: Run Migration

Add the is_app_access column to track app-based users:

php artisan migrate

Step 3: Test It Out

Option A: Browser Test

Visit your authentication page:

https://yoursite.com/game/lucky88_master/app-access

You should see the authentication UI.

Option B: Test with cURL

If your app is whitelisted:

curl -H "X-App-Package: com.jambo.app" \
  https://yoursite.com/game/lucky88_master/app-access

Should auto-grant access and redirect to the game.

✅ Success! Your app access system is now configured. Users can enter games securely without manual token generation.

📱 Using in Your App

Dart/Flutter Example

import 'package:http/http.dart' as http;

Future<void> accessGame() async {
  final response = await http.get(
    Uri.parse('https://yoursite.com/game/lucky88_master/app-access'),
    headers: {
      'X-App-Package': 'com.jambo.app',
    },
  );
  
  if (response.statusCode == 200) {
    // Game session established
    print('Access granted!');
  }
}

🔑 Key Features

  • ✅ Automatic package verification for whitelisted apps
  • ✅ Password authentication for non-whitelisted apps
  • ✅ Beautiful custom authentication UI
  • ✅ Backend-enforced session validation
  • ✅ Zero manual token generation needed

❓ Troubleshooting

Access Denied?

  • Check that your app package is in BD_GAME_FINAL_ALLOWED_APP_PACKAGES
  • Verify the X-App-Package header matches exactly
  • Check if migration has been run: php artisan migrate:status

Page Not Found?

  • Ensure Laravel is running properly
  • Check the game code exists (e.g., lucky88_master)
  • Check routes/bd_game_final.php for the route definition

🎯 দ্রুত শুরু গাইড

মাত্র ৫ মিনিটে অ্যাপ-ভিত্তিক গেম অ্যাক্সেস সিস্টেম চালু করুন!

ধাপ ১: পরিবেশ কনফিগারেশন সম্পাদনা করুন

আপনার .env ফাইল খুলুন এবং যোগ করুন:

BD_GAME_FINAL_ALLOWED_APP_PACKAGES=com.jambo.app,com.example.games
BD_GAME_FINAL_DEVELOPER_PASSWORD=#jambo

প্রতিস্থাপন করুন:

  • com.jambo.app আপনার প্রকৃত অ্যাপ প্যাকেজ নাম দিয়ে
  • #jambo আপনার সুরক্ষিত ডেভেলপার পাসওয়ার্ড দিয়ে

ধাপ ২: মাইগ্রেশন চালান

অ্যাপ-ভিত্তিক ব্যবহারকারীদের ট্র্যাক করার জন্য is_app_access কলাম যোগ করুন:

php artisan migrate

ধাপ ৩: এটি পরীক্ষা করুন

বিকল্প A: ব্রাউজার পরীক্ষা

আপনার প্রমাণীকরণ পৃষ্ঠা পরিদর্শন করুন:

https://আপনারসাইট.com/game/lucky88_master/app-access

আপনি প্রমাণীকরণ ইন্টারফেস দেখতে পাবেন।

বিকল্প B: cURL দিয়ে পরীক্ষা করুন

যদি আপনার অ্যাপ হোয়াইটলিস্টেড হয়:

curl -H "X-App-Package: com.jambo.app" \
  https://আপনারসাইট.com/game/lucky88_master/app-access

স্বয়ংক্রিয়ভাবে অ্যাক্সেস প্রদান করবে এবং গেমে পুনঃনির্দেশ করবে।

✅ সাফল্য! আপনার অ্যাপ অ্যাক্সেস সিস্টেম এখন কনফিগার করা হয়েছে। ব্যবহারকারীরা ম্যানুয়াল টোকেন তৈরি ছাড়াই নিরাপদে গেমে প্রবেশ করতে পারে।

📱 আপনার অ্যাপে ব্যবহার করুন

Dart/Flutter উদাহরণ

import 'package:http/http.dart' as http;

Future<void> accessGame() async {
  final response = await http.get(
    Uri.parse('https://আপনারসাইট.com/game/lucky88_master/app-access'),
    headers: {
      'X-App-Package': 'com.jambo.app',
    },
  );

    // গেম সেশন প্রতিষ্ঠিত
    print('অ্যাক্সেস অনুমোদিত!');
  }
}

❓ সমস্যা সমাধান

অ্যাক্সেস অস্বীকার?

  • নিশ্চিত করুন আপনার অ্যাপ প্যাকেজ BD_GAME_FINAL_ALLOWED_APP_PACKAGES এ আছে
  • X-App-Package হেডার ঠিক মেলে কিনা যাচাই করুন
  • মাইগ্রেশন চালানো হয়েছে কিনা চেক করুন: php artisan migrate:status

পৃষ্ঠা পাওয়া যাচ্ছে না?

  • নিশ্চিত করুন Laravel সঠিকভাবে চলছে
  • গেম কোড বিদ্যমান আছে কিনা চেক করুন (যেমন lucky88_master)
  • routes/bd_game_final.php এ রুট সংজ্ঞা পরীক্ষা করুন
