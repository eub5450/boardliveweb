BD GameFinal cPanel Subdomain Deployment Guide

This guide explains how to upload and run the Laravel game project on a cPanel subdomain such as play.example.com. It includes file placement, database import, environment settings, cron setup, and test steps.

Lobby: /play_bd_game
Game entry: /game/{gameCode}/e
App access: /game/{gameCode}/app-access
API: /game/api/{gameCode}/...

1. What Is Inside The Zip

  • Laravel 10 game runtime.
  • 22 configured GameFinal games.
  • Premium mobile lobby with banner-only game cards.
  • Teen Patti, Lucky 7/77/88, and Fruit Slot UI assets.
  • App access flow for APK/webview clients.
  • Game-related SQL dump in the package database folder.
  • Production verifier command: php artisan bdgamefinal:verify-production.

2. Create The Subdomain

  1. Open cPanel.
  2. Go to Domains or Subdomains.
  3. Create a subdomain, for example play.example.com.
  4. Set the document root to the Laravel public folder if cPanel allows it: /home/USER/play/public.
  5. If cPanel forces a root like /home/USER/public_html/play, upload the project there and point requests to the included public/index.php.

3. Upload Files

  1. Upload the generated project zip to the subdomain folder.
  2. Extract it.
  3. Make sure these paths exist:
    • artisan
    • bootstrap/app.php
    • public/index.php
    • public/game_final_assets
    • storage
  4. Set folder permissions:
    • storage: writable
    • bootstrap/cache: writable

4. Create Database And Import SQL

  1. In cPanel, open MySQL Databases.
  2. Create a database, for example cpuser_play.
  3. Create a database user and password.
  4. Add the user to the database with all privileges.
  5. Open phpMyAdmin, select the database, and import the SQL file from the zip.

The package includes a game-related dump named like:

database/game_final_cpanel.sql

5. Configure .env

Create or edit .env in the project root. Important values:

APP_NAME="BD GameFinal"
APP_ENV=production
APP_DEBUG=false
APP_URL=https://play.example.com

DB_CONNECTION=mysql
DB_HOST=localhost
DB_PORT=3306
DB_DATABASE=cpuser_play
DB_USERNAME=cpuser_playuser
DB_PASSWORD=YOUR_DB_PASSWORD

CACHE_DRIVER=file
SESSION_DRIVER=file
QUEUE_CONNECTION=sync
BROADCAST_DRIVER=log

BD_GAME_FINAL_REALTIME_ENABLED=false
BD_GAME_FINAL_DISPATCH_VIA_QUEUE=false
BD_GAME_FINAL_CACHE_STORE=file
BD_GAME_FINAL_POLL_FALLBACK_MS=1500
BD_GAME_FINAL_ALLOWED_APP_PACKAGES=com.jambo.app,com.example.games
BD_GAME_FINAL_DEVELOPER_PASSWORD=CHANGE_THIS_TO_A_STRONG_24_CHAR_SECRET

Use a strong developer password in production. Do not keep a short test password.

6. Run Laravel Commands

Use cPanel Terminal or SSH:

cd /home/USER/play
php artisan key:generate --force
php artisan optimize:clear
php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan bdgamefinal:verify-production

If Composer dependencies are not included or need refresh:

composer install --no-dev --optimize-autoloader

7. Round Loop / Cron Job

The games need the round loop to advance betting, stop-bet, result, payout, and next-round phases.

Preferred cron command every minute:

* * * * * cd /home/USER/play && php artisan bdgamefinal:tick >> /dev/null 2>&1

If your host supports a long-running process, you can use:

php artisan bdgamefinal:watch --sleep=1

Most shared cPanel hosts do not allow permanent long-running commands, so cron tick is safer.

8. App / WebView Access

For app wrappers, open:

https://play.example.com/game/teen_patti/app-access?app_package=com.jambo.app

If package verification is allowed, the user is redirected into the game automatically. Otherwise, the password page appears.

Lobby URL:

https://play.example.com/play_bd_game

9. Test Checklist

  • Open /play_bd_game on mobile and desktop.
  • Tap each lobby banner and confirm it enters the correct game.
  • Confirm betting countdown appears only during betting.
  • Place a small bet and confirm balance updates.
  • Wait for stop-bet, reveal, winner popup, payout, and next round.
  • Run php artisan bdgamefinal:verify-production.
  • Check storage/logs/laravel.log for errors.

10. Common Problems

401 Unauthorized

Use the approved app-access or lobby-first flow with a valid user session.

500 Server Error

Check storage/logs/laravel.log. Common causes are missing database tables, wrong DB password, unwritable storage, or missing required user columns.

Rounds Not Moving

Add the cron job for php artisan bdgamefinal:tick.

Images Missing

Confirm public/game_final_assets uploaded fully and the subdomain document root points to public.
