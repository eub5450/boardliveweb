📱 App-Based Game Access System

This system provides secure URL-based access to games from mobile/desktop applications without requiring manual token issuance. It includes automatic package verification, password authentication, and backend session validation.

🔑 Key Features

  • Automatic package verification for whitelisted apps
  • Developer password authentication for non-whitelisted apps
  • Custom authentication UI for app access
  • Backend session validation with no public access
  • Dart/Flutter app compatibility

⚙️ Configuration

Step 1: Set Allowed App Packages

Edit your .env file:

# Multiple packages separated by commas
BD_GAME_FINAL_ALLOWED_APP_PACKAGES=com.jambo.app,com.example.games,com.mycompany.app
BD_GAME_FINAL_DEVELOPER_PASSWORD=#jambo

Step 2: App Package Format

Package names should follow standard format:

com.company.appname
org.organization.game
com.example.myapp

🚀 Usage

For Whitelisted Apps

1. Generate Entry URL

https://fairylive.online/game/{gameCode}/app-access?app_package=com.jambo.app

2. Send Request with Package Header (Dart/Flutter)

import 'package:http/http.dart' as http;

final response = await http.get(
  Uri.parse('https://fairylive.online/game/lucky88_master/app-access'),
  headers: {
    'X-App-Package': 'com.jambo.app',
  },
);

3. Response Handling

Success (200): Session established, redirect to game

Unauthorized (401): Package not whitelisted, show password form

Not Found (404): Game code does not exist

For Non-Whitelisted Apps

1. Visit Authentication Page

https://fairylive.online/game/lucky88_master/app-access

2. Show Password Form

If not whitelisted, users see a custom authentication UI prompting for the developer password.

3. Submit Credentials

App sends password with the request:

final response = await http.post(
  Uri.parse('https://fairylive.online/game/lucky88_master/app-access'),
  headers: {
    'X-App-Package': 'com.unknown.app',
    'X-Developer-Password': '#jambo',
  },
);

📊 Request/Response Format

Header Required Description
X-App-Package Yes App package name (e.g., com.jambo.app)
X-Developer-Password If not whitelisted Developer password for non-whitelisted apps
Accept No Set to "application/json" for JSON response

🔐 Security Features

  • Whitelist-based verification: Only approved apps get instant access
  • Password protection: Unauthorized apps can still access if they know the password
  • Session validation: Backend validates every session for additional security
  • No public token exposure: Tokens are never shared in URLs or requests
  • HTTPS required: All requests should use secure connections

💻 API Endpoints

Endpoint Method Purpose
/game/{gameCode}/app-access GET/POST Main app access entry point
/game/{gameCode}/start GET Start-to-play page after authentication
/game/api/{gameCode}/status GET Get game status and current round

🧪 Testing

Test with cURL

# Test whitelisted app
curl -H "X-App-Package: com.jambo.app" \
  https://fairylive.online/game/lucky88_master/app-access

# Test non-whitelisted app
curl -H "X-App-Package: com.unknown.app" \
  -H "X-Developer-Password: #jambo" \
  https://fairylive.online/game/lucky88_master/app-access

Browser Testing

Simply visit the app-access URL in your browser:

https://fairylive.online/game/lucky88_master/app-access
💡 Note: Browser requests without the app package header will show the authentication UI for security purposes.

📱 অ্যাপ-ভিত্তিক গেম অ্যাক্সেস সিস্টেম

এই সিস্টেমটি মোবাইল/ডেস্কটপ অ্যাপ্লিকেশন থেকে গেমগুলিতে নিরাপদ URL-ভিত্তিক অ্যাক্সেস প্রদান করে ম্যানুয়াল টোকেন জারি করার প্রয়োজন ছাড়াই। এতে স্বয়ংক্রিয় প্যাকেজ যাচাইকরণ, পাসওয়ার্ড প্রমাণীকরণ এবং ব্যাকএন্ড সেশন যাচাইকরণ অন্তর্ভুক্ত রয়েছে।

🔑 মূল বৈশিষ্ট্য

  • স্বয়ংক্রিয় প্যাকেজ যাচাইকরণ হোয়াইটলিস্টেড অ্যাপের জন্য
  • ডেভেলপার পাসওয়ার্ড প্রমাণীকরণ অ-হোয়াইটলিস্টেড অ্যাপের জন্য
  • কাস্টম প্রমাণীকরণ UI অ্যাপ অ্যাক্সেসের জন্য
  • ব্যাকএন্ড সেশন যাচাইকরণ কোন পাবলিক অ্যাক্সেস ছাড়াই
  • Dart/Flutter অ্যাপ সামঞ্জস্য

com.company.appname

import 'package:http/http.dart' as http;

final response = await http.get(

৩. প্রতিক্রিয়া পরিচালনা

সাফল্য (200): সেশন প্রতিষ্ঠিত, গেমে পুনঃনির্দেশ করুন

অননুমোদিত (401): প্যাকেজ হোয়াইটলিস্টেড নয়, পাসওয়ার্ড ফর্ম দেখান

পাওয়া যায়নি (404): গেম কোড বিদ্যমান নেই

অ-হোয়াইটলিস্টেড অ্যাপের জন্য

১. প্রমাণীকরণ পৃষ্ঠা পরিদর্শন করুন

https://fairylive.online/game/lucky88_master/app-access

२. পাসওয়ার্ড ফর্ম দেখান

যদি হোয়াইটলিস্টেড না হয়, ব্যবহারকারীরা ডেভেলপার পাসওয়ার্ড চাইছে এমন কাস্টম প্রমাণীকরণ UI দেখেন।

३. শংসাপত্র জমা দিন

অ্যাপ অনুরোধের সাথে পাসওয়ার্ড পাঠায়:

final response = await http.post(
  Uri.parse('https://fairylive.online/game/lucky88_master/app-access'),
  headers: {
    'X-App-Package': 'com.unknown.app',
    'X-Developer-Password': '#jambo',
  },
);

📊 অনুরোধ/প্রতিক্রিয়া ফরম্যাট

হেডার প্রয়োজনীয় বর্ণনা
X-App-Package হ্যাঁ অ্যাপ প্যাকেজ নাম (যেমন com.jambo.app)
X-Developer-Password যদি হোয়াইটলিস্টেড না হয় অ-হোয়াইটলিস্টেড অ্যাপের জন্য ডেভেলপার পাসওয়ার্ড
Accept না JSON প্রতিক্রিয়ার জন্য "application/json" এ সেট করুন

🔐 নিরাপত্তা বৈশিষ্ট্য

  • হোয়াইটলিস্ট-ভিত্তিক যাচাইকরণ: শুধুমাত্র অনুমোদিত অ্যাপ তাৎক্ষণিক অ্যাক্সেস পায়
  • পাসওয়ার্ড সুরক্ষা: অননুমোদিত অ্যাপ পাসওয়ার্ড জানলেও অ্যাক্সেস করতে পারে
  • সেশন যাচাইকরণ: ব্যাকএন্ড অতিরিক্ত নিরাপত্তার জন্য প্রতিটি সেশন যাচাই করে
  • কোন পাবলিক টোকেন এক্সপোজার নেই: টোকেন কখনও URL বা অনুরোধে শেয়ার করা হয় না
  • HTTPS প্রয়োজন: সমস্ত অনুরোধ নিরাপদ সংযোগ ব্যবহার করা উচিত

💻 API এন্ডপয়েন্ট

এন্ডপয়েন্ট পদ্ধতি উদ্দেশ্য
